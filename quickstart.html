1. Before Wednesday — Get Access & Install Tools

Do these before you arrive. The less setup we do on Wednesday morning, the more time we have to build.

  • Request GitHub access — Go to Freshservice catalog. You need Member access (not Enterprise) to the GitHub-Klaviyo org. Create a GitHub account with your Klaviyo email first at github.com/join if you don't have one. Should be instant approval.
  • Request Cursor access — Go to Freshservice catalog and submit the request. This is instant approval.
  • Request Claude Code access — You need "Claude Premium" access. If you don't have it, ask in #hackathon-knowledge-transfer-agent and Kioja will help.
  • Download and install Cursor — cursor.com — it's a code editor that has AI built in. Think of it like a Google Doc but for code, where you can talk to an AI assistant while you work.
  • Install Node.js and Git — Don't Google around for this. Just open Cursor's AI chat and paste the prompt below. It'll walk you through everything.
I need to get my computer set up for a Next.js hackathon project. Check if I have Node.js (version 18 or higher) and Git installed. If anything is missing, walk me through installing it step by step. I'm on a Mac.

If any of this feels confusing, don't worry. Just install Cursor, and we'll sort the rest out Wednesday morning.

2. Getting Started with Cursor

Cursor is the main tool most of you will use. It looks like a code editor, but the magic is the AI chat panel — you describe what you want in English and it writes the code for you.

1 Open Cursor

Launch Cursor after installing it. It'll look like a code editor with a sidebar on the left showing files and folders.

2 Connect Cursor to Claude

By default Cursor uses its own AI, but we want it to use Claude (which is smarter for our use case). Open the AI chat panel and type:

I want to set Claude as my default AI model in Cursor. Walk me through the settings to switch from the default model to Claude. Show me step by step where to click.
What this does: Tells Cursor to use Anthropic's Claude model instead of the default. Claude is what powers our entire project, so using it in your editor means everything speaks the same language.

3 Open the project folder

Once Kioja shares the GitHub link Wednesday morning, you'll need to open the project. Go to File > Open Folder and navigate to wherever you cloned the project (we'll cover cloning in the next section).

4 Use the AI chat

Press Cmd+L (Mac) or Ctrl+L (Windows) to open the AI chat panel. This is where you talk to the AI. You can:

- Ask it to explain code you're looking at
- Tell it to build something and it writes the code
- Paste errors and ask it to fix them
- Highlight code, then ask the AI to modify just that part

5 Use the built-in terminal

Cursor has a terminal built in at the bottom of the screen. Press Ctrl+` (backtick key, above Tab) to toggle it. This is where you'll run Claude Code commands if you want to use both tools together.

The key thing to remember: Cursor is where you SEE the code and files. The AI chat panel is where you TALK to the AI. You describe what you want, it writes the code, and you can see it appear in the editor. If something looks wrong, just tell it.

3. Getting Started with Claude Code

Claude Code is Anthropic's command-line AI tool. It runs in your terminal (the black window with text). It's powerful because it can read your entire project, make changes across multiple files, and run commands — all from a conversation.

1 Install Claude Code

Open Terminal (or the terminal inside Cursor) and paste this:

npm install -g @anthropic-ai/claude-code
What this does: Downloads and installs Claude Code on your computer so you can use it from any terminal window.

2 Navigate to the project folder

In your terminal, navigate to where you cloned the project. If you're not sure where it is, paste this prompt into Claude Code:

Help me find and navigate to the hackathon project folder on my computer. It should be a folder called something like "knowledge-transfer" that I cloned from GitHub.

3 Start Claude Code

Once you're in the project folder, just type:

claude
What this does: Launches the Claude Code AI assistant. It reads your entire project and is ready for you to tell it what to build. You type in English, it writes code.

4 Planning mode

Before you ask Claude Code to build anything, hit Shift+Tab to enter planning mode. This makes the AI think through the approach and ask you questions before writing any code. It prevents the AI from going in the wrong direction.

You can use Cursor and Claude Code together. Use Cursor to SEE your files visually and make quick edits via the chat panel. Use Claude Code in the Cursor terminal for bigger tasks where you want the AI to work across multiple files at once.

4. Git in Plain English

You do not need to learn Git. You just need to understand 5 concepts so you know when to ask the AI to do something. Here they are:

Clone = Download the project

When Kioja shares the GitHub link, you need to download the project to your computer. That's called "cloning." You do this once, at the very beginning. After that, the project lives on your machine and you work on it locally.

Branch = Your own workspace

Think of the project like a shared Google Doc. A branch is your own private copy where you can make changes without affecting anyone else's work. Everyone works on their own branch. When you're done, your changes get merged back into the main version. This is how 7 people work on the same project without stepping on each other.

Commit = Save your progress

A commit is like hitting Ctrl+S, but smarter — it saves a snapshot of your work with a note about what you changed. If something breaks later, you can go back to any previous commit. Think of it as save points in a video game. Commit early, commit often.

Push = Upload your saves to the cloud

Your commits live on your computer until you push them. Pushing uploads your saved work to GitHub (the cloud) so the team can see it and Kioja can wire it into the main project. If your laptop dies, your pushed work is safe.

Pull Request (PR) = "Hey, my work is ready"

When you're done with your piece, a pull request is how you say "I'm finished, please add my work to the main project." It packages up everything you did on your branch and sends it to Kioja to review and merge. You don't need to know how to create one — just tell the AI.

5. The Git Prompts

Here are the exact prompts to paste into Cursor's AI chat or Claude Code. You don't need to memorize any commands. Just know when to use each prompt.

Important

These prompts work in both Cursor's AI chat panel (Cmd+L) and in Claude Code (the terminal). Pick whichever you're more comfortable with.

When: Wednesday morning, first thing

What you're doing: Downloading the project to your computer for the first time.

Clone the hackathon project from GitHub. The repo URL is [Kioja will share this]. Put it somewhere sensible on my computer like my Documents or Desktop folder. Then open it.

Kioja will share the exact GitHub URL in Slack Wednesday morning. Just paste it where it says [Kioja will share this].

When: Right after cloning, before you start building

What you're doing: Creating your own private workspace so your changes don't affect anyone else.

Create a new branch for my work. I'm working on [YOUR ROLE — e.g., "the chat UI", "the interview questions", "the landing page"]. Name the branch something like feature/[short-description]. Switch me to that branch so I'm working on it.
Why this matters: Without a branch, everyone would be editing the same files at the same time and overwriting each other's work. Your branch is your safe space. Break things, experiment, try stuff — it only affects your branch.

When: Every 30-60 minutes, or whenever something is working

What you're doing: Saving a snapshot of your progress. Do this often — especially when something works.

Commit my current changes. Here's what I did: [describe briefly — e.g., "got the chat bubbles working", "finished the question tree for topics 1-4", "added the progress bar"]
Why this matters: If you break something in the next hour, you can always go back to this save point. The more often you commit, the less work you lose if something goes wrong. Think of it as quicksave.

Rule of thumb: If you just got something working and you think "nice, that works" — that's the moment to commit. Don't wait until everything is done.

When: After a few commits, or before you take a break

What you're doing: Uploading your saved work to the cloud (GitHub) so the team can see it and it's backed up.

Push my branch to GitHub so the team can see my work.
Why this matters: Until you push, your work only exists on your laptop. Pushing backs it up to GitHub and lets Kioja see your progress. Push at least once an hour, and always before stepping away from your computer.

When: If Kioja says "I pushed updates" or "pull the latest"

What you're doing: Downloading the latest changes that Kioja or someone else made to the main project.

Pull the latest changes from the main branch and merge them into my branch. If there are any conflicts, help me resolve them.
Why this matters: As Kioja wires everyone's pieces together, the main project evolves. Pulling keeps your branch up to date so you're building on the latest version, not an outdated one.

When: Something broke and you want to go back

What you're doing: Going back to the last working version.

Something broke. Undo my last changes and go back to the last working version. Show me what changed so I can figure out what went wrong.
Why this matters: This is your undo button. Because you committed often (right?), the AI can take you back to any save point. No work is permanently lost.

When: You're done with your piece and it's ready for Kioja

What you're doing: Packaging up your finished work and sending it to Kioja to add to the main project.

I'm done with my work on [YOUR ROLE]. Create a pull request to merge my branch into main. Describe what I built and what it does.
Why this matters: This is how your finished work gets delivered. Kioja reviews it, and once it looks good, it gets merged into the main project. You don't need to know the mechanics — the AI handles all of it.

6. Day-Of: Your First 15 Minutes

Here's exactly what to do when you sit down Wednesday morning after the team brief.

1 Open Cursor

Launch Cursor on your machine.

2 Clone the project

Open the terminal in Cursor (press Ctrl+`) and paste the clone prompt from section 5 above with the GitHub link Kioja shares.

3 Open the project folder in Cursor

Go to File > Open Folder and select the folder you just cloned.

4 Create your branch

In the AI chat (Cmd+L) or terminal, paste the branch prompt from section 5 with your role name.

5 Find your folder

In the file sidebar on the left, look for your role's folder. Kioja will have set up the directory structure so everyone has their own space:

Interview Designer → /content/interview/
Doc Template → /content/templates/ and /features/synthesis/
Landing Page → /app/(marketing)/
Chat UI → /features/chat/
Glean Integration → /features/glean/
Confluence Publisher → /features/confluence/

6 Start building

Open the AI chat and start describing what you want. Look at your role card in the Playbook for the "START HERE" section — that's your first task.

That's it. You're up and running. From here, just follow your role card in the playbook. Remember to commit every 30-60 minutes and push before breaks.

7. When You're Stuck

Something broke and you don't know why

I'm getting this error: [paste the exact error]. What's wrong and how do I fix it?

Copy and paste the exact error message. Don't describe it from memory — the specific text matters.

You don't understand what the AI did

Explain what you just did in plain English. I'm not a developer — tell me what each file does and why you made those changes.

The AI seems to be going in circles

Stop. You've tried this approach 3 times and it's not working. Try a completely different approach to solve this. Search the web to verify your assumptions.

If the AI can't fix something in 3 tries, it's stuck in a loop. See tip #3 in the playbook.

You want to see what the app looks like

How do I run this project locally so I can see it in my browser?

You want to install a package the AI told you about

Install [package name] and add it to the project. Show me how to use it for [what you need].

You've been stuck for more than 15 minutes

Post in #hackathon-knowledge-transfer-agent. Seriously. There are no dumb questions, and someone on the team has probably hit the same thing. Kioja can usually unblock you in 5 minutes.

The cheat sheet

Clone = Download the project (once, at the start)
Branch = Create your own workspace (once, after cloning)
Commit = Save your progress (every 30-60 min)
Push = Upload saves to GitHub (every hour, before breaks)
Pull = Get latest updates from the team (when Kioja says to)
PR = "I'm done, here's my work" (once, when finished)
Undo = Go back to the last save point (when things break)
